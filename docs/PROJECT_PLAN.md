# Agent Workflow Bench Plan

## Positioning

This should become a public, recruiter-readable systems project rather than a loose collection of helper scripts.

The value proposition is:

- multi-agent workflows are trendy
- few public repos evaluate them rigorously
- even fewer connect them to coding and high-trust workflow use cases

## Phase 1

Ship a tiny but honest benchmark.

- 6 to 10 tasks total
- 3 workflow variants
- 1 coding domain plus 1 document or evaluation domain
- JSON run artifacts
- Markdown summary report

## Phase 2

Add stronger evidence.

- failure taxonomy
- per-role ablations
- human takeover logging
- cost and latency dashboards
- regression suite for prompt or routing changes

## Phase 3

Turn it into a durable portfolio anchor.

- public case study
- reproducible benchmark command
- screenshots and sample reports
- one honest conclusion section on where multi-agent helps and where it is overhead
